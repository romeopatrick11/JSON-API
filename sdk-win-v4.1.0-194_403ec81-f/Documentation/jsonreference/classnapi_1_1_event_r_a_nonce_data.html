Detailed Description

Notification of a fresh nonce to be signed by the Roaming Authenticator service.

Every time the Roaming Authentication process is initiated a new nonce is generated by the Nymi Band. This is providing the nonce to the NEA. This should be provided to the Roaming Authenticator service.

NOTE: Do not reuse nonces.

NOTE: Do not assume that you will recieve this after receiving the response to initiating the Roaming Authentication response (see RoamingAuthResp). These messages are sent on different NAPI threads and there is no ordering guaranteed (or required).

Details of Roaming Authentication, including the partner public key, are described in the Roaming Authenticator section of the SDK documentation.
