    Overview of NAPI

    The Nymi API (NAPI) is an asynchronous API that works by an interchange of JSON strings between a Nymi-Enabled app (NEA) and NAPI.

    Making a request to NAPI (napi::put) is a non-blocking call returning immediately, and it always succeeds if NAPI has been started (see napi::PutOutcome for details). Responses to requests are obtained by calling napi::get.

    Note: This success of napi::put even after napi::terminate has been called is an historical detail that will be addressed in a future release.

    There are three common approaches to writing an NEA using NAPI:

    • Intersperse calls to napi::put and napi::get.
    • Run napi::put and napi::get in their own threads, and link the response to the request using the exchange.
    • Run napi::put and napi::get in their own threads, and dispatch the responses by the exchange.

    The first approach is most useful in simple or short running NEAs. The second, and especially third, approaches are more useful in longer running or complex NEAs.

    NAPI Functions

    NAPI consists of four functions:

    • napi::configure - Configure and start NAPI.
    • napi::get - Returns a mix of JSON responses and NAPI-generated notifications. Use the exchange attribute to match responses to requests.
    • napi::put - Send a JSON message to NAPI.
    • napi::terminate - Quit NAPI. The NEA must call this function before exiting.

    Note: If there are operations in progress when terminate() is called, there may be a delay of up to 30 seconds before it completes.

    NEAs on Windows OS **must** call napi::terminate() before exiting. If terminate() is not called before the NEA exits, the dynamic libraries will crash on unload.

    The NAPI JSON Examples section provides examples of JSON requests to and responses from NAPI.

    NAPI JSON Envelope Objects

    Request Object

    The envelope wrapping the NAPI JSON request is patterned as follows:

    napi::RequestEnvelope{
      public:
        string     path;   // The path of the request
        string     exchange;   // The exchange identifier
        Json::Value     request;   // The actual request object
      };

    When a request is made to NAPI it is wrapped in this JSON object. The response from NAPI is wrapped in the same object with some additional attributes defined.

    The exchange attribute is an arbitrary string determined by the NEA that can be used to match this request to its response.

    Response Object

    The envelope wrapping the NAPI JSON response is patterned as follows:

    napi::ResponseEnvelope : public RequestEnvelope{
      public:
        bool     completed;   // Whether the request completed
        bool     successful;   // Whether the request was successful
        Json::Value     response;   // The actual response object
        vector< vector< string, string > > errors;   // Any errors reported by NAPI
      };

    When a response from NAPI is received, it is wrapped in the request JSON object, with the additional attributes described here.
