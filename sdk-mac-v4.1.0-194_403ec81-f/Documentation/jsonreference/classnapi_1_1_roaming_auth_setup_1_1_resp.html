Detailed Description

Response to setup Roaming Authentication.

This response contains key and key-ID information required for a Roaming Authenticator service to work. This information, along with other user and NEA-specific information (for example, email address or other credentials), should be forwarded to the roamining authenticator service when the response indicates a successful outcome.

Many NEAs will revoke the provision used to setup Roaming Authentication upon a successful outcome.

Details of Roaming Authentication, including the partner public key, are described in the Roaming Authentication section of the SDK documentation.
