Detailed Description

onFoundChange notification

See the detailed discussion of the 'found' status.

The notification will contain and associate pid only if it is known. It is possible that on first appearance and after provisioning the pid is unknown for a short time (< 1s) and some notifications may be reported without the pid.

NOTE: Until either this or the onPresenceChange notification is received with a pid specified, the NEA must not assume that any provision is present (especially just after provisioning is complete).

Possible Found States:

Enumerator
undetected 

The Nymi Band is undetected (will normally happen when the Nymi Band 'walks away')

unclasped 

The Nymi Band has been unclasped (will normally happen after the Nymi Band has been authenticated)

unprovisionable 

The Nymi Band cannot be provisioned, normally because the Nymi Band is fully provisioned (by other NEAs)

anonymous 

The Nymi Band has not been provisioned by this NEA (this can occur momentarily with a provisioned Nymi Band when it is first detected)

discovered 

The Nymi Band has entered provisioning mode.

provisioning 

The Nymi Band is in the process of being provisioned by this NEA.

identified 

The Nymi Band is claiming to be a provisioned band but this has not been confirmed (NAPI will attempt to confirm)

authenticated 

The Nymi Band is confirmed to have been provisioned.
