    Developer Quick Start

    Use Nymi’s SDK to develop Nymi-enabled applications (NEAs) for use with the Nymi Band. The SDK includes the Nymi API (NAPI), and the Nymulator (Nymi Band Simulator) to test your code without the need for a physical Nymi Band.

    The Nymi API 4.1 is provided as a dynamic library on Windows, and a framework installer on macOS. To demonstrate the functionality exposed through NAPI, we have provided a sample application (apy), which demonstrates how to provision a Nymi Band, how to obtain cryptographic assets from a provisioned Nymi Band, and several other functions. The sample app is an NEA written in Python.

    This Developer Quick Start walks you through the setup of the Python sample app, the initialization of NAPI, and the provisioning process. From there, you can investigate the provided code to explore the other functions.

    Development with the Nymulator

    You do not need a physical Nymi Band for NEA development. You can start developing immediately with the Nymulator, which simulates up to six Nymi Bands in a software environment.

    The Nymulator is supported on the same OS X and Windows OS versions as the SDK.

    This Quick Start describes how to get started with the Nymi API and the Nymulator. We recommend following this process before working with a physical Nymi Band.

    The Development with a Nymi Band section describes the configuration changes required for running NAPI and the apy sample app against a physical Nymi Band.

    Developing on Windows Platforms

    1. Download the SDK zip. The link below is a direct download. All files are available on GitHub®.
      • Windows SDK
    2. The Nymulator requires the Visual Studio Redistributable 2013 x86. If you do not already have the redistributable package installed, run sdk4.1\Nymulator\vcredist_x86.exe.

    3. Clone or fork the apy sample app from the Nymi/SampleApps/Python GitHub repository. The link below opens in a new tab.
      • Sample app

    4. Copy the SDK zip file into the same directory as the sample app.

    5. Start the Nymulator. The Nymulator must be running before you start the sample app, otherwise NAPI cannot be properly initalized.
      1. Double-click the executable: sdk4.1\Nymulator\Nymulator.exe.
      2. Make a note of the port number that the Nymulator is using. By default this is port 9088. If this port is in use by another process, the Nymulator decrements the number and tries again until it finds an open port.

    6. Open a command window in the sample app directory.

    7. At the command prompt, type: python go.py.

    8. When prompted, type y to allow the sample app to copy required files.
      The sample app is designed to prompt you for missing information and provide feedback on options where applicable. For example, if you try to run python go.py before starting the Nymulator, the following information is printed to the command window:

      Checking connectivity… done.
      Failed to start, attempt: 1
      Failed to start, attempt: 2
      Failed to start, attempt: 3
      Failed to start, attempt: 4
      Failed to start, attempt: 5
      Maybe check that the Nymulator is running.

    9. In the Nymulator, click Add to create a virtual Nymi Band. When they are created, virtual Nymi Bands are authenticated, but not provisioned.

    10. Provision the Nymi Band. Provisioning is a one-time process that establishes a trusted relationship between the Nymi Band and a specific NEA running on a specific device. The Nymi Band initiates the process by generating a pattern that appears on the Nymi Band’s LEDs. This pattern is also computed independently by NAPI and sent to the NEA. The user wearing the Nymi Band must confirm that the patterns match.

      The pattern appears on the Nymi Band graphic in the upper-right corner of the Nymulator. At the same time, the pattern also appears in the command window, where a plus sign (+) indicates an LED is ON, and a minus sign (-) indicates an LED is OFF. If the two patterns match, you accept the pattern as described in this procedure.

      1. In the Nymulator, click Begin Provisioning. A pattern appears on the Nymi Band graphic.
      2. In the command window, the following message appears:
        found a band to provision; pattern is, followed by a series of 5 plus and/or minus signs
      3. Confirm that the pattern in the command window matches the one displayed in the Nymulator.
      4. Type y to accept the pattern match.

    Success messages appear in the command window, and in the Nymulator Activity Log.

    Note: An NEA must have explicit user agreement to provision a Nymi Band. An NEA must provide some way for the user to confirm the LED pattern displayed on their Nymi Band. The NEA can either: display the pattern to the user, and ask for confirmation that the displayed pattern matches that on the band, or ask the user to input the pattern on their band, and proceed if it matches the pattern sent to the NEA by NAPI. The first approach is more convenient for the user, while the second provides greater certainty.

    Congratulations! You have successfully provisioned a virtual Nymi Band.

    Now that the Nymi Band is provisioned, you can verify its authentication status, generate and obtain cryptographic assests, and more. These features are described in the NAPI JSON Examples section.

    To start working with a physical Nymi Band, see the Development with a Nymi Band section, which describes the configuration changes required for running NAPI and the sample app against a physical Nymi Band.

    Development with a Nymi Band

    This section assumes that you have worked through the Development with the Nymulator section for your platform, and therefore you already have the SDK and sample app downloaded and configured. Remember that Nymi-enabled apps (NEAs) communicate only with authenticated Nymi Bands. Use the Nymi Band app to authenticate the Nymi Band before you use the sample app.

    To work with a physical Nymi Band, follow the appropriate procedure below.

    For Windows OS:

    1. Shut down the Nymulator.

    2. Plug in the Bluetooth adapter (included in the box with your Nymi Band).

    3. Install and enable the Nymi Bluetooth Service (included in sdk4.1\NymiBluetoothService\).

    4. Quit any running instances of apy.

    5. Run python go.py --real. The --real parameter tells the sample app to look for a Nymi Band instead of the Nymulator.

    6. Tap the top sensor on your Nymi Band continuously until you see the provisioning pattern (that is, the lights flash in an outward pattern, from the center out to both sides). You can expect to see the pattern after four taps.

    7. Provision your Nymi Band.
      1. Confirm that the light pattern in the command window matches the one displayed on your Nymi Band.
      2. In the command window, type y.

    For macOS / OS X:

    1. Shut down the Nymulator.

    2. Quit any running instances of apy.

    3. Run python go.py --real. The --real parameter tells the sample app to look for a Nymi Band instead of the Nymulator.

    4. Tap the top sensor on your Nymi Band continuously until you see the provisioning pattern (that is, the lights flash in an outward pattern, from the center out to both sides). You can expect to see the pattern after four taps.

    5. Provision your Nymi Band.
      1. Confirm that the light pattern in the console matches the one displayed on your Nymi Band.
      2. In the console, type y.
    Congratulations! You have successfully provisioned a physical Nymi Band.
